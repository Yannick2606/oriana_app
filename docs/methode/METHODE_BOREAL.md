# P-004 — Méthode BORÉAL

> **Statut :** Vivant  
> **Nature :** Référentiel méthodologique  
> **Nom :** « BORÉAL » est un nom de travail provisoire  
> **Révision :** toute évolution significative doit être tracée dans l’architecture des décisions

## 1. Finalité

La Méthode BORÉAL décrit la manière dont orIAna transforme une situation métier en compréhension partagée, puis en décision, en action et en connaissance réutilisable.

Elle vise à éviter trois dérives :

- agir avant d’avoir compris ;
- confondre volume d’activité et création de valeur ;
- perdre l’expérience acquise au fil des missions, des échanges et des décisions.

La méthode place l’humain au centre. L’intelligence artificielle assiste l’analyse, la recherche, la formalisation et la projection, mais elle ne se substitue ni au jugement professionnel ni à la responsabilité de la décision.

## 2. Principes directeurs

La méthode repose sur les principes suivants :

1. partir du réel et du besoin exprimé ;
2. distinguer les faits, les hypothèses, les perceptions et les intentions ;
3. rechercher la valeur avant de choisir une solution ;
4. rendre les raisonnements explicables et traçables ;
5. mobiliser les référentiels sans enfermer l’analyse ;
6. projeter les conséquences à court, moyen et long terme ;
7. organiser la décision au bon niveau de responsabilité ;
8. transformer chaque action en apprentissage ;
9. capitaliser les connaissances utiles ;
10. améliorer continuellement les pratiques, les modèles et les outils.

## 3. Cycle BORÉAL

Le cycle méthodologique comprend sept étapes complémentaires.

### 3.1 Observer

Recueillir la situation telle qu’elle existe réellement :

- contexte ;
- acteurs ;
- données disponibles ;
- contraintes ;
- événements ;
- signaux faibles ;
- irritants ;
- attentes explicites et implicites.

L’observation doit conserver la source, la date, le niveau de fiabilité et les éventuelles limites de l’information.

### 3.2 Qualifier

Transformer les éléments recueillis en objets compréhensibles :

- faits établis ;
- informations à confirmer ;
- hypothèses ;
- risques ;
- opportunités ;
- dépendances ;
- contradictions ;
- niveau d’urgence ;
- niveau d’importance.

La qualification évite qu’une intuition, une préférence ou une donnée non vérifiée soit traitée comme un fait.

### 3.3 Comprendre

Reconstituer les causes, les relations et les dynamiques de la situation.

Cette étape doit notamment répondre aux questions suivantes :

- Quel problème cherche-t-on réellement à résoudre ?
- Pour qui ce problème crée-t-il une difficulté ou une perte de valeur ?
- Quelles sont les causes immédiates et profondes ?
- Quels comportements, processus, règles, outils ou données influencent la situation ?
- Quelles conséquences apparaissent si rien ne change ?

### 3.4 Évaluer la valeur

Analyser la valeur produite, détruite, protégée ou rendue possible.

La valeur ne se limite pas au gain financier. Elle peut être :

- économique ;
- commerciale ;
- opérationnelle ;
- patrimoniale ;
- humaine ;
- informationnelle ;
- réglementaire ;
- environnementale ;
- réputationnelle ;
- stratégique.

L’évaluation doit également identifier les bénéficiaires, les coûts, les efforts, les risques et les externalités.

### 3.5 Projeter

Construire plusieurs futurs possibles plutôt qu’une réponse unique présentée comme certaine.

La projection peut intégrer :

- scénario de continuité ;
- scénario prudent ;
- scénario cible ;
- scénario dégradé ;
- conditions de réussite ;
- points de rupture ;
- impacts sur les acteurs, les données, les processus et les actifs.

Les hypothèses utilisées doivent rester visibles et révisables.

### 3.6 Décider et agir

La décision doit préciser :

- le résultat attendu ;
- le décideur ;
- les contributeurs ;
- le responsable de l’exécution ;
- les moyens ;
- les échéances ;
- les indicateurs ;
- les risques acceptés ;
- les conditions de réexamen.

L’action est organisée au niveau le plus pertinent, selon les droits, les responsabilités et le principe de subsidiarité.

### 3.7 Apprendre et capitaliser

Chaque action significative doit pouvoir enrichir le patrimoine d’orIAna.

La capitalisation porte notamment sur :

- les décisions prises ;
- leurs motivations ;
- les résultats constatés ;
- les écarts ;
- les enseignements ;
- les pratiques efficaces ;
- les erreurs à ne pas reproduire ;
- les connaissances réutilisables ;
- les référentiels à faire évoluer.

Le retour d’expérience clôt un cycle et nourrit le suivant.

## 4. Les objets de la méthode

La méthode s’applique à différents objets métier :

- organisation ;
- entreprise ;
- client ;
- actif immobilier ;
- site ;
- bâtiment ;
- cellule ;
- lot ;
- terrain ;
- mandat ;
- opportunité ;
- projet ;
- décision ;
- risque ;
- connaissance ;
- relation entre acteurs.

Un même parcours peut être commencé sur le terrain depuis un smartphone, puis poursuivi ou finalisé au bureau sur un ordinateur. La continuité du contexte, des données et des justificatifs doit être préservée.

## 5. Cercle d’impact

Toute analyse doit examiner les effets au-delà de l’objet immédiatement concerné.

Le cercle d’impact comprend selon les cas :

- l’utilisateur ;
- l’équipe ;
- l’agence ;
- le client ;
- les partenaires ;
- les occupants ;
- le territoire ;
- les actifs immobiliers ;
- l’organisation ;
- les parties prenantes externes ;
- le patrimoine de données et de connaissances.

Cette approche évite les optimisations locales qui dégradent la valeur globale.

## 6. Référentiels et expérience

Les référentiels fournissent une base commune pour qualifier, comparer et décider. Ils peuvent couvrir :

- typologies d’actifs ;
- usages ;
- secteurs géographiques ;
- risques ;
- maturité ;
- critères de valeur ;
- règles réglementaires ;
- bonnes pratiques ;
- indicateurs ;
- modèles de décisions.

Les référentiels ne doivent jamais être considérés comme immuables. L’expérience réelle, lorsqu’elle est documentée et validée, peut conduire à leur évolution.

## 7. Rôle de l’intelligence artificielle

L’intelligence artificielle peut :

- synthétiser des informations ;
- détecter des incohérences ;
- rapprocher des situations comparables ;
- proposer des hypothèses ;
- identifier des risques ou opportunités ;
- préparer des scénarios ;
- suggérer des actions ;
- faciliter la formalisation et la capitalisation.

Elle doit cependant :

- expliciter les sources et hypothèses disponibles ;
- signaler l’incertitude ;
- respecter les droits d’accès et la confidentialité ;
- permettre la contestation et la correction ;
- conserver l’humain comme autorité de décision ;
- laisser une trace exploitable des raisonnements déterminants.

## 8. Gouvernance de la méthode

La Méthode BORÉAL est un document vivant, mais son évolution doit rester gouvernée.

Toute modification structurante doit préciser :

- le besoin à l’origine de la modification ;
- les éléments d’expérience qui la justifient ;
- les impacts sur les processus, les données, l’architecture et les utilisateurs ;
- les documents associés à mettre à jour ;
- la décision de validation.

Les variantes métier sont autorisées à condition de rester compatibles avec l’ADN, la Charte et la Constitution d’orIAna.

## 9. Critères d’une application réussie

La méthode est correctement appliquée lorsque :

- le problème réel est clairement formulé ;
- les faits et hypothèses sont distingués ;
- la valeur attendue est identifiée ;
- les impacts sont examinés ;
- la décision est explicable ;
- les responsabilités sont claires ;
- les résultats sont mesurables ;
- l’expérience est capitalisée ;
- les connaissances deviennent réutilisables ;
- le dispositif reste simple pour l’utilisateur.

## 10. Positionnement

La Méthode BORÉAL n’est ni une procédure rigide ni un simple modèle documentaire.

Elle constitue un cadre commun de raisonnement et d’action permettant à orIAna de relier :

- l’expérience du terrain ;
- les données ;
- la connaissance ;
- la décision ;
- l’intelligence humaine ;
- l’intelligence artificielle ;
- la création durable de valeur.

Son nom définitif pourra évoluer. Sa finalité demeure : **mieux comprendre pour mieux décider, mieux agir et mieux transmettre.**
