# Architecture des décisions orIAna

## Objet

Ce document conserve les décisions structurantes du projet orIAna, leur contexte, les alternatives étudiées et leurs impacts.

Il complète l'historique des décisions en expliquant le raisonnement ayant conduit aux choix d'architecture, de produit et de gouvernance.

## Format d'une décision

Chaque décision importante suit le modèle suivant :

- Date
- Contexte
- Problème à résoudre
- Alternatives étudiées
- Décision retenue
- Raisons du choix
- Conséquences
- Documents impactés

## Principes

- Une décision importante doit être explicable.
- Les alternatives pertinentes doivent être conservées.
- Les impacts futurs doivent être identifiés.
- Les décisions peuvent évoluer, mais leur historique doit rester accessible.
